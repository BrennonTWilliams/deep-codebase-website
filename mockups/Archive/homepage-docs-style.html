Overview

You know the drill: need their authentication middleware for your project, spend hours digging through files trying to extract the implementation and dependencies. What should be a quick extraction turns into reverse-engineering their entire architecture.

‚úì The Solution
Deep-codebase finds, extracts, and prepares components for integration‚Äîautomatically. Point our CLI at any repository with your target feature and get extraction-ready code with full dependency chains, usage examples, and integration guidance.

Combines Claude Sonnet 4 AI analysis with Tree-sitter structural parsing for accurate, context-aware code extraction from any language-supported codebase.

Quick Start

Get started with deep-codebase in under 5 minutes. Install via pip and start extracting components from any repository.

Install deep-codebase

Requires Python 3.8+ and Claude CLI installed on your system.

terminal 
pip install deep-codebase

Run AI analysis

Query any repository with natural language to get architectural insights and component locations.

terminal 
deep-codebase analyze https://github.com/user/repo \
  "How does authentication work?"

Extract with code snippets

Add --include-snippets flag to extract actual code with dependencies mapped and ready for integration.

terminal 
deep-codebase analyze https://github.com/user/repo \
  "Extract the caching module" \
  --include-snippets 12

Core Features

Deep-codebase combines advanced static analysis, semantic understanding, and AI-powered insights to deliver production-ready code extraction.

Intelligent Code Selection

Hybrid TF-IDF semantic analysis combined with call graph dependency tracking automatically identifies the most relevant code snippets for your query. Not all code is equally relevant‚Äîour system ranks results by architectural significance and query relevance.

Algorithm: TF-IDF + Call Graph Analysis

Smart Query Understanding

Intent detection adapts results based on what you're asking. "How does authentication work?" gives broad architectural context. "How to use the auth API?" provides focused integration examples. "Extract the caching module" delivers complete dependencies.

Modes: Architecture | Integration | Extraction | Feature

Tree-Sitter Static Analysis

Understanding Intent Detection

The system automatically detects your query intent and optimizes results accordingly. Different intents produce different analysis depths and extraction strategies.

Intent Type Example Query Analysis Focus
Architecture "How does authentication work?" Broad architectural context, system design patterns, component relationships
Integration "How to use the auth API?" Public interfaces, usage patterns, API endpoints, integration examples
Extraction "Extract the caching module" Complete component boundaries, dependency chains, all related code
Feature "Where is rate limiting implemented?" Precise feature location, focused results, minimal extra context

Command Reference

Complete CLI command reference with all available options and flags.

Basic Usage 
deep-codebase analyze <repository-url> "<your-query>" [options]

Arguments

Argument Description Required
repository-url GitHub repository URL or local path to codebase Yes
query Natural language query describing what to extract or analyze Yes

Options

Flag Description Default
--include-snippets N Extract N code snippets with byte-accurate source code 0 (disabled)
--output DIR Specify custom output directory for extraction results ./extracted/
--format FORMAT Output format: markdown, json, or html markdown
--verbose Enable detailed logging and progress information false
